## 🎓 Testing Patterns Used

### Pattern 1: Component Mounting
```javascript
test('should render without crashing', () => {
  render(<Button>Click Me</Button>);
  expect(screen.getByText('Click Me')).toBeInTheDocument();
});
```

### Pattern 2: User Interaction
```javascript
test('should call onClick handler when clicked', async () => {
  const handleClick = jest.fn();
  render(<Button onClick={handleClick}>Click</Button>);
  await userEvent.click(screen.getByRole('button'));
  expect(handleClick).toHaveBeenCalledTimes(1);
});
```

### Pattern 3: Controlled Component State
```javascript
test('should update input value on keystroke', async () => {
  const TestComponent = () => {
    const [value, setValue] = useState('');
    return <Input value={value} onChange={(e) => setValue(e.target.value)} />;
  };
  render(<TestComponent />);
  const input = screen.getByRole('textbox');
  await userEvent.type(input, 'Hello');
  expect(input).toHaveValue('Hello');
});
```

### Pattern 4: Error States
```javascript
test('should display error when validation fails', async () => {
  render(<Input error="This field is required" />);
  expect(screen.getByText('This field is required')).toBeInTheDocument();
});
```
